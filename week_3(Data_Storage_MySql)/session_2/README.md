# Session 2: MySQL Data Storage Tasks

This folder contains two Python scripts that demonstrate advanced MySQL database operations using the `mysql-connector-python` library.

## Prerequisites

- Python 3.x
- MySQL server installed and running
- Required Python packages: `mysql-connector-python`, `python-dotenv`
- A `.env` file in the project root with the following variables:
  - `DB_HOST`: Your MySQL host (e.g., `localhost`)
  - `DB_USER`: Your MySQL username
  - `DB_PASSWORD`: Your MySQL password
  - `POSTS_API_URL`: For Task 2, set to `https://jsonplaceholder.typicode.com/posts`

## Task 1: Multi-Table Relational System (`task1.py`)

**Difficulty: Hard**

This script creates a relational database system for an online store.

### What it does:
- Creates a database called `store_db` with three tables:
  - `customers` (customer_id, name, city)
  - `products` (product_id, name, price)
  - `orders` (order_id, customer_id, product_id, quantity) with foreign key constraints
- Inserts sample data: 10 customers, 8 products, and 20 orders
- Runs analytical queries and prints results:
  - Total money spent per customer (sorted highest first)
  - Most ordered product by total quantity
  - Customers who placed more than 2 orders
  - Average order value per city
- Exports the revenue-per-customer data to `revenue_report.csv`

### Deliverables:
- MySQL `store_db` database
- `revenue_report.csv` file


## Task 2: API Monitor with Change Detection (`task2.py`)

**Difficulty: Hard**

This script monitors an external API for changes and logs them in a database.

### What it does:
- Creates a database called `monitor_db` with two tables:
  - `posts` (id, userId, title, body) - stores fetched API data
  - `change_log` (log_id, post_id, change_type, changed_at) - tracks changes
- Fetches posts from JSONPlaceholder API
- Detects new posts and modifications between runs
- Logs changes with timestamps
- Runs analytical queries and prints results:
  - Post count per user
  - All change log entries from the latest run
  - User who triggered the most change events

### Special Instructions:
- On first run, all posts are inserted and logged as 'NEW'
- To test change detection: manually update a post in MySQL (e.g., `UPDATE posts SET title='Changed' WHERE id=1`) and re-run the script
- The script will detect the mismatch and log it as 'MODIFIED'

### Deliverables:
- MySQL `monitor_db` database with populated tables
