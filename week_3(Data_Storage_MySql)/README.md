# Week 3 — Data Storage with MySQL

This folder contains the Week 3 internship tasks for building MySQL-backed data workflows using Python. The work includes database creation, API ingestion, query analysis, data export, and a summary report.

## Included Tasks

- `task1.py`
  - Creates `library_db` and a `books` table with sample book records.
  - Adds a linked `reviews` table.
  - Runs queries for books published after 2000, Fiction books with rating > 4.0, average ratings, and book count by genre.

- `task2.py`
  - Creates `app_db` and tables for `users` and `posts`.
  - Fetches users and posts from JSONPlaceholder API.
  - Extracts nested JSON fields and inserts users plus posts for user IDs 1, 2, and 3.
  - Runs queries for users sorted by name, cities with multiple users, and a user-post join.

- `task3.py`
  - Creates `weather_db` and a `forecasts` table.
  - Fetches 7-day weather data for New York, London, and Tokyo from Open-Meteo.
  - Stores max/min temperatures and average humidity.
  - Runs queries for the hottest city, the hottest day, and days with a temperature range > 10°C.
  - Writes a summary report to `summary.txt`.

- `task4.py`
  - Creates `grades_db` and a `students` table.
  - Inserts student scores, assigns grades, updates grades, and adds a `passed` boolean column.
  - Deletes students who scored below 50.
  - Queries student counts by grade.

- `task5.py`
  - Creates `news_db` and an `articles` table.
  - Fetches news for India, Nepal, and the USA.
  - Stores article metadata in MySQL and exports the results to `news_data.csv`.

- `task5_analysis.py`
  - Connects to `news_db` and reads the stored articles.
  - Runs analysis queries on country counts, top sources, latest articles, languages, and time range.
  - Saves the analysis output to `summary.txt`.

## Setup

1. Install dependencies:
   - `python -m pip install mysql-connector-python python-dotenv requests pandas`

2. Create a `.env` file in this folder containing MySQL and API configuration values:

```env
DB_HOST=your_mysql_host
DB_USER=your_mysql_user
DB_PASSWORD=your_mysql_password
API_URL=https://jsonplaceholder.typicode.com/users
POSTS_API_URL=https://jsonplaceholder.typicode.com/posts
WEATHER_API_URL=https://api.open-meteo.com/v1/forecast
BASE_URL=<news_api_endpoint>
API_KEY=<your_news_api_key>
```

3. Run any task file:
   - `python task1.py`
   - `python task2.py`
   - `python task3.py`
   - `python task4.py`
   - `python task5.py`

## Notes

- Each script is designed to be self-contained and build its own database.
- `task3.py` writes the weather summary to `summary.txt`.
- `task5.py` exports MySQL query results to `news_data.csv`.
- `task5_analysis.py` reads the stored news data and writes analysis results to `summary.txt`.
- Use a valid MySQL server and correct `.env` credentials before running the scripts.

## Lessons Learned

- Database design: created multiple MySQL databases and tables with relationships, unique keys, and constraints.
- API ingestion: fetched data from public APIs and inserted nested JSON fields into SQL tables.
- Query writing: used `GROUP BY`, `JOIN`, `ORDER BY`, `HAVING`, and aggregate functions for analysis.
- Data export: exported query results to CSV and generated text summaries.
- Error handling: managed database connection issues, duplicate inserts, and API request failures.
- Modular coding: built reusable functions for fetching, storing, and reporting data.
